# mlops-hello 👋

My **first project** on the journey to becoming an MLOps engineer.

This repo is the hands-on result of **Lesson 02 — First Python + Git Workflow**, where I learned the core developer loop every MLOps engineer uses daily:

> **write code → `git add` → `git commit` → `git push`**

---

## What's inside

| File | Purpose |
|------|---------|
| `hello.py` | A tiny Python script with a `greet()` function. |
| `.gitignore` | Tells Git which files to never commit (secrets, data, temp files). |
| `README.md` | This file — describes the project. |

---

## Run it

```bash
python hello.py
```

**Output:**

```text
Hello, Sohan! Welcome to MLOps.
```

---

## What I learned

- Git's 3 areas: **working directory → staging → local repo → GitHub**.
- The difference between `git add` (stage), `git commit` (save snapshot), and `git push` (share).
- Why `.gitignore` matters — never commit secrets or large data files.
- Connecting a local repo to GitHub over **SSH**.
